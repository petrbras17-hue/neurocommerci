# NEURO COMMENTING — Scrum Plan v2: AI Marketing Agency as a Service

Дата: 2026-03-12
Цель: полностью рабочая платформа нейрокомментинга за 1 месяц до первых платящих клиентов

## Визия продукта

Клиент даёт ссылку на свой продукт — платформа силами сотни ИИ-агентов:
- Анализирует рынок и находит целевые каналы
- Создаёт профили аккаунтов под бренд клиента
- Генерирует контекстные AI-комментарии
- Ведёт статистику и даёт анализ ROI
- Всё автоматизировано — от закупки аккаунтов до отчётов

## Технические параметры цели

| Параметр | Значение |
|----------|----------|
| Рабочих аккаунтов | 100+ |
| Каналов на аккаунт | 50+ |
| Комментариев в день на аккаунт | 30+ |
| Языки | RU (потом мультиязычный) |
| Прокси | Статические резидентные (Webshare/IPRoyal), 1:1 с аккаунтами |
| Аккаунты | Прокачанные Session+JSON, номера +77 (KZ) / +998 (UZ) |
| Бюджет инфраструктуры | ~$275/мес на 100 аккаунтов |

## Критические лимиты Telegram (из исследования)

| Параметр | Лимит | Источник |
|----------|-------|---------|
| Комментариев в час на аккаунт | **12-14 макс** | spark.ru, vc.ru |
| Задержка между комментами | **350-400 сек (~6 мин)** | blackhatworld, практика |
| Вступление в каналы | **5-6 за 20-30 мин** | telegramtools |
| Макс каналов на аккаунт | **500** | Telegram лимит |
| ЛС незнакомым в день | **50** | практика |
| Перерыв (молодой аккаунт) | После **50-150** комментов | практика |
| Перерыв (прокачанный) | После **250-360** комментов | практика |
| Прогрев до ready | **14 дней** (мин 5-7) | partnerkin, telegramtools |

**Пересчёт**: 30 комментов/день на аккаунт = ~5 часов работы с задержками. Это безопасный режим.
При 100 аккаунтах = **3,000 комментов/день** = ~90 заявок = 3-6 продаж/день.

## Что уже реализовано (Sprint 1-6)

- FastAPI + PostgreSQL + Redis + Docker Compose
- Tenant isolation (RLS), JWT auth, email/password + Telegram
- React/Vite web shell с 20+ страницами
- AI Router (Gemini + OpenRouter) с boss/manager/worker тирами
- Farm Orchestrator + Farm Thread state machine
- Channel Parser Service
- Profile Factory (AI генерация + apply)
- Anti-Detection (3 режима: conservative/moderate/aggressive)
- Health Scorer + Quarantine Manager
- Warmup Engine
- Mass Reactions, Neuro Chatting, Neuro Dialogs
- User Parser, Folder Manager
- Smart Commenter + Channel Indexer
- Unified Job Worker для 11 очередей
- API endpoints для всех модулей
- Auth sessions management
- Account monitor + appeal scripts

## Что НЕ реализовано / требует доработки

1. **Реальное Telethon-подключение** — код есть, но не протестирован с живыми аккаунтами на VPS
2. **Proxy Management** — нет UI для управления прокси, нет auto-health-check
3. **Автозакупка аккаунтов/прокси** — нет интеграции с маркетплейсами
4. **Клиентский onboarding** — нет flow "дай ссылку на продукт → получи комментинг"
5. **Биллинг** — архитектура готова, нужны ключи Stripe/YooKassa
6. **Аналитика ROI** — endpoints есть, нужны реальные данные
7. **Мониторинг в реальном времени** — нет live dashboard с WebSocket
8. **CAPTCHA автоматизация** — скрипт есть, нужна интеграция в платформу
9. **Масштабирование VPS** — один сервер, нужна готовность к 100+ потокам

## SPRINT 7: Proxy & Account Management UI (3-4 дня) — БЕЗ АККАУНТОВ

**Цель**: Полный UI для управления прокси и аккаунтами — готовность к загрузке

### Задачи:

**7.1 Proxy Management Page** (frontend + backend)
- `/app/proxies` — полная страница управления прокси
- `POST /v1/proxies/bulk-import` — массовый импорт из файла (host:port:user:pass)
- `GET /v1/proxies` — список с статусами (alive/dead/bound/free)
- `GET /v1/proxies/health` — проверка живости всех прокси
- `POST /v1/proxies/cleanup` — удаление мёртвых
- Кнопка "Проверить все" + cron каждые 30 мин
- Уведомление в Нейросводку если живых прокси < 20%

**7.2 Account Manager Page v2** (frontend + backend)
- `/app/accounts` — расширенная страница
- Массовый импорт .session + .json (drag & drop, ZIP)
- Массовая привязка прокси (auto-assign свободный прокси)
- Статусы: free / warmup / farming / frozen / appeal / banned / dead
- Timeline каждого аккаунта (история событий)
- Bulk actions: warmup all, start farm, stop farm

**7.3 Account Lifecycle Integration** (backend)
- Интеграция `account_monitor.py` в unified worker (cron каждые 30 мин)
- Auto-appeal при обнаружении frozen (если cooldown прошёл)
- Уведомления о смене статуса в Нейросводку

**7.4 Bulk Onboarding Flow** (готовность)
- Загрузка N аккаунтов + N прокси за один раз
- Автоматическая привязка 1:1
- Автоматический запуск warmup (conservative, 48 часов)
- Автоматический переход в farm после warmup
- Прогресс-бар на Dashboard

### Definition of Done:
- [ ] Proxy management UI полностью работает
- [ ] Account management UI с bulk import готов
- [ ] Account monitor интегрирован в worker
- [ ] Bulk onboarding flow готов к загрузке реальных аккаунтов
- [ ] Тесты проходят

## SPRINT 8: Smart Commenting & Channel Intelligence (3-4 дня) — БЕЗ АККАУНТОВ

**Цель**: AI-комментарии неотличимы от живых, каналы подбираются автоматически

### Задачи:

**8.1 Smart Commenter v2** (backend)
- Интеграция `smart_commenter.py` в farm thread
- Контекстный анализ поста перед комментированием
- Emoji-first trick (сначала эмодзи, потом редактируем в текст — обходит антиспам)
- Never-first rule (ждём 2-3 живых комментария перед нашим)
- Вариативность: 10+ шаблонов стилей комментирования
- Лимиты из исследования: 350-400 сек между комментами, макс 12-14/час
- A/B тестирование стилей (какой получает больше реакций)

**8.2 Channel Intelligence** (backend)
- Автоматический подбор каналов по нише клиента
- Spam score каждого канала
- Анализ аудитории канала (язык, активность, тематика)
- Blacklist каналов с агрессивной модерацией
- Автообновление базы каналов

**8.3 Comment Quality Dashboard** (frontend)
- Показ сгенерированных комментариев (preview mode)
- Статистика: комментарии/день, реакции, ответы
- Фильтр по каналу, аккаунту, дате

**8.4 Anti-Detection v2** (backend)
- Рандомизация online-статуса
- Имитация чтения между комментариями
- Случайные реакции на чужие посты
- Разные интервалы для каждого аккаунта (антибан из исследования)

### Definition of Done:
- [ ] Smart Commenter v2 генерирует контекстные комментарии
- [ ] Channel Intelligence подбирает каналы по нише
- [ ] Anti-Detection v2 реализует все антибан правила
- [ ] Comment dashboard работает

## SPRINT 9: Client Onboarding — "Ссылка → Комментинг" (3-4 дня) — БЕЗ АККАУНТОВ

**Цель**: Клиент даёт ссылку на продукт — платформа делает всё сама

### Задачи:

**9.1 Product Analyzer** (backend, AI)
- Клиент вводит URL продукта
- AI анализирует: что за продукт, ЦА, тон бренда, УТП
- Формирует brief: ключевые слова, стиль комментариев, целевые каналы
- Сохраняет в `product_briefs` таблицу

**9.2 Auto Campaign Creator** (backend)
- На основе brief автоматически:
  - Подбирает 50+ каналов по нише
  - Назначает аккаунты (из пула)
  - Генерирует профили аккаунтов под бренд
  - Создаёт шаблоны комментариев
  - Запускает farm

**9.3 Client Dashboard** (frontend)
- `/app/campaigns/{id}` — клиент видит:
  - Сколько комментариев сегодня
  - В каких каналах
  - Примеры комментариев
  - Статистика: просмотры, реакции, переходы

**9.4 Onboarding Wizard** (frontend)
- Шаг 1: Вставь ссылку на продукт
- Шаг 2: AI анализирует и показывает brief (редактируемый)
- Шаг 3: Выбери план (кол-во комментариев/день)
- Шаг 4: Оплата (или триал 3 дня)
- Шаг 5: Запуск!

### Definition of Done:
- [ ] Клиент может пройти весь onboarding за 5 минут
- [ ] AI brief точно определяет нишу и тон
- [ ] Campaign dashboard работает
- [ ] Wizard полностью функционален

## SPRINT 10: Analytics & ROI Dashboard (3-4 дня) — БЕЗ АККАУНТОВ

**Цель**: Клиент видит результат и платит с удовольствием

### Задачи:

**10.1 Analytics Pipeline** (backend)
- Трекинг: комментарии, реакции на них, ответы, упоминания
- Расчёт: CPL (cost per lead), CTR комментариев, конверсия
- Сравнение каналов: где лучше всего работает
- Хранение: `analytics_events` таблица + Redis кэш

**10.2 ROI Dashboard** (frontend)
- Графики: комментарии/день, реакции/день, рост подписчиков
- Тепловая карта: лучшее время для комментирования
- Сравнение: этот месяц vs прошлый
- Рекомендации AI: "Увеличьте активность в канале X — там CTR 4.2%"

**10.3 Weekly Report** (автоматический)
- AI генерирует еженедельный отчёт
- Отправка клиенту в Telegram / email
- Ключевые метрики + рекомендации

**10.4 Admin Analytics** (internal)
- Аккаунты: живы/мертвы по платформе
- Расход AI токенов
- Revenue / MRR / Churn rate

### Definition of Done:
- [ ] Analytics pipeline сохраняет все события
- [ ] ROI dashboard показывает графики
- [ ] Weekly report генерируется
- [ ] Admin видит бизнес-метрики

## SPRINT 11: Self-Healing & Auto-Purchasing Logic (3-4 дня) — БЕЗ АККАУНТОВ

**Цель**: Логика автоматического восстановления и закупки (без реальных покупок)

### Задачи:

**11.1 Self-Healing Engine** (backend)
- При бане аккаунта → автозамена из пула
- При смерти прокси → автозамена + перепривязка
- При FloodWait → автокарантин + перераспределение каналов
- При freeze → авто-апелляция (если cooldown прошёл)

**11.2 Auto-Purchase Framework** (backend)
- API интеграция с Webshare (прокси)
- API интеграция с GrizzlySMS / DarkStore (аккаунты) — подготовка
- Автозакупка при < 10% свободных ресурсов
- Уведомление админу + подтверждение

**11.3 Platform Health Dashboard** (frontend)
- Зелёный/жёлтый/красный статус платформы
- Предсказание: "Через 3 дня закончатся аккаунты"
- Telegram-уведомления для всех критических событий

### Definition of Done:
- [ ] Self-healing логика работает (тестируется mock-ами)
- [ ] Auto-purchase API подключены
- [ ] Health dashboard показывает статус платформы
